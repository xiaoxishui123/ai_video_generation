## 🎯 核心优化：火山方舟配音时长匹配

### 问题描述
使用火山方舟 Seedance 1.5 Pro 的智能时长 + 音频生成功能时，视频时长无法正确匹配旁白长度。

例如：48字旁白（约12秒），但生成的视频只有5秒。

### 根因分析
官网体验中心的输入文本**既是视频内容也是配音内容**，模型根据输入文本长度决定视频时长。

而我们之前的实现把 narration 附加在 prompt 后面：
```
{prompt}

【配音旁白】{narration}
```

模型可能只根据前面的 prompt 计算时长，而忽略了配音旁白的长度。

### 修复方案
调整 prompt 构建顺序，把 narration 放在最前面：
```
{narration}

【画面风格】{prompt}
```

这样模型会优先理解配音内容和所需时长。

### 影响范围
- 火山方舟 Seedance 1.5 Pro 的文生视频和图生视频功能
- 当 `enable_audio: True` 且有 `narration` 参数时生效

### 使用建议
1. 使用 `duration_mode: smart`（智能时长）
2. 开启 `enable_audio`
3. 传递 `narration` 参数（旁白文本）
4. 视频时长会自动匹配旁白长度
